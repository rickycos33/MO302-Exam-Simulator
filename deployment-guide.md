# Deployment Guide for MO302 Exam Simulator

This guide explains how to deploy the HTML output generated by opalAI to GitHub Pages.

## Step 1: Prepare Your HTML File
Ensure that you have generated the `exam.html` file using opalAI and it's ready to be deployed.

## Step 2: Push the HTML File to GitHub
1. Navigate to your repository on GitHub: [MO302 Exam Simulator](https://github.com/rickycos33/MO302-Exam-Simulator).
2. Click on the `Add file` button and select `Upload files`.
3. Drag your `exam.html` file into the upload interface or use the file selector to choose it.
4. Scroll down to the Commit section, add a commit message (e.g., "Add exam.html for GitHub Pages deployment"), and click the `Commit changes` button to upload your file.

## Step 3: Enable GitHub Pages
1. Go to the repository settings by clicking on the `Settings` tab.
2. In the left sidebar, click on `Pages`.
3. Under `Source`, select the branch you want to use for GitHub Pages (usually `main`) and the root directory, then click `Save`.
4. After a few moments, GitHub will provide a URL for your GitHub Pages site. 

## Step 4: Access Your Exam
Once GitHub Pages is enabled, you can access your exam at the following URL:

```
https://<your-username>.github.io/MO302-Exam-Simulator/exam.html
```

Replace `<your-username>` with `rickycos33` to access the deployed HTML output directly.

## Conclusion
You have successfully deployed the `exam.html` file to GitHub Pages. You can share the provided link with others to access the exam online.
